# UI方案选择

本文件用于记录 agent蜂群 的前端 UI 方向选择。

原型目录：

```text
F:\ai共同体(知识库)\20_项目\agent蜂群\ui-prototypes
```

入口文件：

```text
ui-prototypes/index.html
```

## 方案 01：控制台型

文件：

```text
ui-prototypes/console.html
```

特点：

- 信息密度高。
- 像真实 SaaS 工作台。
- 适合长期使用。
- 任务、Agent、成本、代码执行确认都能放在一屏。

适合：

```text
正式产品主界面
```

推荐程度：

```text
高
```

## 方案 02：流程图型

文件：

```text
ui-prototypes/flow.html
```

特点：

- 强调 Agent 自动编排流程。
- 容易看懂系统怎么跑。
- 适合解释“架构师 → 调度器 → 多 Agent → 审查”。

适合：

```text
项目创建后的流程运行页
新手引导页
流程可视化页面
```

推荐程度：

```text
中高
```

## 方案 03：IDE 型

文件：

```text
ui-prototypes/ide.html
```

特点：

- 像代码编辑器。
- 强调本地 Runner、修改计划、审查输出。
- 适合后续“用户确认后自动改代码”。

适合：

```text
Runner 执行页
代码修改确认页
审查与 diff 页面
```

推荐程度：

```text
高
```

## 方案 04：指挥中心型

文件：

```text
ui-prototypes/command.html
```

特点：

- 视觉更强。
- 适合展示多模型蜂群状态。
- 更像演示大屏或愿景页面。

适合：

```text
演示页
全局运行态势页
团队汇报页
```

推荐程度：

```text
中
```

## 初步建议

建议不要只选一套，而是组合：

```text
主工作台：方案 01 控制台型
流程运行页：方案 02 流程图型
Runner 执行确认页：方案 03 IDE 型
展示/总览页：方案 04 指挥中心型
```

这样产品会更完整：

- 平时干活用控制台。
- 看 Agent 自动流程用流程图。
- 改代码确认用 IDE 型页面。
- 给别人展示用指挥中心。
