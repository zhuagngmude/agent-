# agent蜂群新窗口交接说明

日期：2026-06-08

## 一句话项目定位

`agent蜂群` 是一个多 AI 智能体协作调度系统，不是普通聊天工具。目标是把一个大项目拆成架构、前端、后端、测试、文档、审查等角色任务，并通过审批、状态管理、知识库和本地 Runner 逐步实现可控的自动协作。

当前阶段先做本地电脑端 Web App，后续再接本地 Runner，最后再考虑封装成桌面应用。

## 如何启动

在项目根目录运行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\start-dev.ps1
```

启动后：

- Mock API: `http://127.0.0.1:8787`
- Web App: `apps\web\index.html`

如果只想启动 API：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\start-mock-api.ps1
```

## 重要约定

1. 每次重要讨论后改文档，然后 commit。
2. 每实现一个功能，commit。
3. 每修一个 bug，commit。
4. 每次大改前，先 commit 当前状态。
5. 工作文件尽量放在 F 盘项目目录，不要放到 C 盘。
6. 真实密钥、个人配置、日志、运行时状态不要提交进 Git。
7. `design/image2/` 暂时不要动。
8. `_internal/` 是本地内部资料目录，已被 `.gitignore` 忽略；AI 不要提交它。
9. 当前是 Web App 优先，不要急着封装桌面 exe。
10. Runner 未来可以读写文件，但必须经过审批、Git checkpoint、任务锁、文件所有权和审查合并机制。
11. Approval Service 和 Runner 必须分离，Runner 不能自己决定执行权限。

## 下一步建议

下一步最合适做：

```text
把任务管理接入 Mock API 和本地状态保存
```

目标：

1. 任务页不只是静态展示。
2. 支持任务状态变化：
   - `queued`
   - `running`
   - `blocked`
   - `waiting_user`
   - `completed`
   - `failed`
   - `cancelled`
3. 支持最小操作：
   - 开始任务
   - 标记完成
   - 标记失败
   - 取消任务
4. 把任务状态也写入 `data/mock/runtime-state.json`。
5. 页面操作后刷新 Dashboard 的活跃任务数量。

推荐新增接口：

```text
POST /api/tasks/:taskId/start
POST /api/tasks/:taskId/complete
POST /api/tasks/:taskId/fail
POST /api/tasks/:taskId/cancel
```

实现顺序：

1. 在 `services/api/mock-data.js` 为任务加初始运行状态快照和 reset。
2. 在 `services/api/server.js` 让 runtime state 保存 tasks。
3. 新增任务 action API。
4. 在 `apps/web/index.html` 的任务页增加任务详情和操作按钮。
5. 在 `apps/web/app.js` 接 API 并刷新页面。
6. 更新 `下一步开发路线.md`。
7. 跑 `node --check` 和接口测试。
8. commit。
