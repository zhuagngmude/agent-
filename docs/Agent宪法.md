# Agent 宪法（项目 AI 协作总规则）

> 本文件为 agent-swarm 项目长期有效的 AI 行为准则。每次进入项目时，必须先读本文件，再读 `docs/AI开发细则.md` 和 `dev-docs/` 下的开发、交接文档。

宪法只保留最高优先级规则；前端、数据库、Git、文档同步等执行细则见 [AI开发细则.md](./AI开发细则.md)。

## 一、边界与禁区

### 受保护路径

以下目录和文件不要修改、不要删除、不要作为自动化写入目标：

- `design/image2/`
- `_internal/`
- `data/mock/runtime-state.json`
- `data/local/`
- `logs/`
- `.playwright-cli/`

### 总规则

1. **禁止擅自添加新依赖**：Rust 侧（`Cargo.toml`）、前端侧（`package.json`）引入新库必须先说明理由并获批准。
2. **禁止破坏公共接口**：Tauri commands、共享 UI 组件、数据库 schema 的签名和行为一旦改动，必须同步更新所有调用方。
3. **禁止绕过 Migration 改库**：任何数据库结构变更必须从 Migration 文件发起，并说明原因和回滚方式。
4. **禁止绕过 Model Gateway**：UI、Agent、Runner 或任务状态机不得绕过 Model Gateway 直接调用 provider。
5. **禁止越过当前阶段边界**：阶段 34 已开放极小范围真实 Runner（仅临时沙箱路径+固定 Git 只读命令+人工二次确认），所有 Runner 执行必须经完整闸门链（preflight→gate→dry-run→exec_lock→minimal_run）；不开放批量/自动 Runner、自由命令、Git commit/push、文件删除、网络请求、云同步、完整权限系统。阶段 35 已开放受控真实模型调用（仅经 Model Gateway、受控模型目录、脱敏审计链路的人工二次确认 project_plan_generation）；不开放前端自由模型配置、key、base URL、prompt、header、body。阶段 35.1 已收口前端中文化。阶段 36 已定义全自动模式宪法与权限边界（`full_auto` 须显式开启、默认 `manual`；总控智能体可自动授权 L0-L3 但不得绕过审计；不开放自动 L4 执行；一键停止和自动降级为必备能力）。不得写入保护路径，不得绕过审批写用户项目文件。

## 二、行为准则

6. **先理解，再动手**：修改任何文件前，必须先阅读相关模块的现有代码和文档，明确其设计意图。
7. **只用最稳妥的方案**：优先选择对现有结构影响最小、最容易验证的实现路径，禁止用临时补丁掩盖底层问题。
8. **无证据不修改**：如果无法从代码、文档或对话记录中找到确凿依据，不得凭猜测修改业务逻辑。
9. **所有风险操作保留后路**：数据库结构变更、目录重组、删除文件、大范围重构前，必须说明回滚方案并确认已有 Git 存档。
10. **用户可见文案默认中文**：前端 UI、提示、按钮、标签、错误信息等用户可见文案默认使用中文；后端枚举值、数据库字段名、内部状态不得直接裸露。英文只允许保留在模型 ID、命令、环境变量、Provider、路径、数据库 ID 等技术标识中。

## 三、决策优先级

当不同目标发生冲突时，按以下顺序决策：

- **架构一致性 > 功能实现速度**
- **数据安全与完整性 > 用户体验便利**
- **代码可维护性 > 代码简洁性**
- **明确的已有规范 > 模型自由偏好**

## 四、完成标准与错误分级

声称"完成"一项任务时，必须满足以下条件：

11. **给出可验证的证明**：提供测试步骤、终端输出或截图，证明功能符合预期。
12. **不留下已知隐患**：无新增错误、警告或明显性能下降；已处理空数据、权限不足等必要边界。
13. **交付物完整**：涉及接口、UI、数据库、阶段状态或 AI 边界的变更，必须同步对应文档和类型。
14. **主动声明未完成项**：任何暂未处理或需人工确认的部分，必须在回复开头以"⚠️ 待确认"明确列出。
15. **禁止选项剧场**：用户要求方案时给出一个推荐主线；备选方案只用于解释为什么不选。

审查和实现时必须将问题分级（详见 `docs/AI开发细则.md §八`）：
- **阻断问题**：破坏核心功能/安全/数据/构建/验收，当轮必须收掉。
- **设计风险**：不会立即破坏但会导致架构漂移，必须说明取舍。
- **可记录债务**：不影响本轮，可暂缓，必须说明不处理原因。
- **无关优化**：不进本轮范围，禁止借机扩大改造。

## 附则：规则运作

- 本宪法规则文件由 AI 工具在每次会话启动时自动加载。
- 项目过程中如果发现规则缺失导致系统性错误，以"补充案"形式追加，但宪法主规则控制在 15 条以内。
- 所有公开计划、交接说明和阶段记录统一放到 `dev-docs/`，不放在项目根目录。

### 全自动模式补充说明（阶段 36）

以下为阶段 36 已定义的全自动模式宪法与权限边界，不作为独立编号规则，但后续阶段实现前必须检查：

- 全自动模式（`full_auto`）必须用户显式开启，默认 `manual`；不得在用户不知情或未确认的情况下自动进入。
- 总控智能体在 `full_auto` 模式下可按自动授权策略为角色智能体授予 L0-L3 权限，但所有授权、模型调用和执行准备必须写入审计记录；不得绕过 Model Gateway、Runner 闸门链。
- 阶段 36 不开放自动 L4 执行；L4（最小真实执行）永远需人工二次确认。
- 一键停止是全自动模式必备能力；任意安全校验失败、审计写入失败或 scope violation 触发自动降级（`full_auto → paused` 或 `full_auto → stopped`）。
- 角色智能体不得自扩权、不得自行选择未启用模型、不得绕过审计直接写文件或执行命令。
- Git commit/push、文件删除、网络请求、自由命令、保护路径写入永不自动。
- 旧阶段 36（角色模型绑定）和旧阶段 37（Skill 绑定）已并入新阶段 40。
