# Agent 宪法（项目 AI 协作总规则）

> 本文件为 agent-swarm 项目长期有效的 AI 行为准则。每次进入项目时，必须先读本文件，再读 `docs/AI开发细则.md` 和 `dev-docs/` 下的开发、交接文档。

宪法只保留最高优先级规则；前端、数据库、Git、文档同步等执行细则见 [AI开发细则.md](./AI开发细则.md)。

## 一、边界与禁区

### 受保护路径

以下目录和文件不要修改、不要删除、不要作为自动化写入目标：

- `design/image2/`
- `_internal/`
- `data/mock/runtime-state.json`
- `data/local/`
- `logs/`
- `.playwright-cli/`

### 总规则

1. **禁止擅自添加新依赖**：Rust 侧（`Cargo.toml`）、前端侧（`package.json`）引入新库必须先说明理由并获批准。
2. **禁止破坏公共接口**：Tauri commands、共享 UI 组件、数据库 schema 的签名和行为一旦改动，必须同步更新所有调用方。
3. **禁止绕过 Migration 改库**：任何数据库结构变更必须从 Migration 文件发起，并说明原因和回滚方式。
4. **禁止绕过 Model Gateway**：UI、Agent、Runner 或任务状态机不得绕过 Model Gateway 直接调用 provider。
5. **禁止越过当前阶段边界**：当前不开放真实 Runner、真实模型调用、云同步、完整权限系统；不得写入保护路径，不得绕过审批写用户项目文件。

## 二、行为准则

6. **先理解，再动手**：修改任何文件前，必须先阅读相关模块的现有代码和文档，明确其设计意图。
7. **只用最稳妥的方案**：优先选择对现有结构影响最小、最容易验证的实现路径，禁止用临时补丁掩盖底层问题。
8. **无证据不修改**：如果无法从代码、文档或对话记录中找到确凿依据，不得凭猜测修改业务逻辑。
9. **所有风险操作保留后路**：数据库结构变更、目录重组、删除文件、大范围重构前，必须说明回滚方案并确认已有 Git 存档。

## 三、决策优先级

当不同目标发生冲突时，按以下顺序决策：

- **架构一致性 > 功能实现速度**
- **数据安全与完整性 > 用户体验便利**
- **代码可维护性 > 代码简洁性**
- **明确的已有规范 > 模型自由偏好**

## 四、完成标准

声称“完成”一项任务时，必须满足以下条件：

10. **给出可验证的证明**：提供测试步骤、终端输出或截图，证明功能符合预期。
11. **不留下已知隐患**：无新增错误、警告或明显性能下降；已处理空数据、权限不足等必要边界。
12. **交付物完整**：涉及接口、UI、数据库、阶段状态或 AI 边界的变更，必须同步对应文档和类型。
13. **主动声明未完成项**：任何暂未处理或需人工确认的部分，必须在回复开头以“⚠️ 待确认”明确列出。

## 附则：规则运作

- 本宪法规则文件由 AI 工具在每次会话启动时自动加载。
- 项目过程中如果发现规则缺失导致系统性错误，以“补充案”形式追加，但宪法主规则控制在 15 条以内。
- 所有公开计划、交接说明和阶段记录统一放到 `dev-docs/`，不放在项目根目录。
