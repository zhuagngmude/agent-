# 验收检查切片

## 1. 验收步骤

### 1.1 功能验收
- [ ] 登录页面可正常访问，URL为 `/login`
- [ ] 输入正确的用户名和密码后，可成功登录并跳转到首页
- [ ] 输入错误的用户名或密码，页面显示错误提示信息
- [ ] 未登录状态下访问受保护页面，自动跳转到登录页
- [ ] 登录成功后，页面显示用户信息（如用户名）

### 1.2 界面验收
- [ ] 页面布局符合设计稿，包含用户名输入框、密码输入框、登录按钮
- [ ] 输入框有对应的标签和占位符提示
- [ ] 登录按钮在点击时有加载状态反馈
- [ ] 错误提示信息样式醒目（红色文字或警告框）
- [ ] 页面在不同屏幕尺寸下响应式良好

### 1.3 安全验收
- [ ] 密码输入框类型为 `password`，输入内容不可见
- [ ] 登录接口使用 HTTPS 协议
- [ ] 登录失败时，不泄露具体是用户名错误还是密码错误
- [ ] 登录接口有防暴力破解机制（如验证码、限流）
- [ ] Token/会话信息存储在 HttpOnly Cookie 中

## 2. 禁止路径

### 2.1 代码层面
- ❌ 禁止在代码中硬编码用户名和密码
- ❌ 禁止将密码以明文形式存储或传输
- ❌ 禁止在前端进行密码验证（必须由后端验证）
- ❌ 禁止使用 `eval()` 或类似危险函数处理用户输入
- ❌ 禁止在 URL 参数中传递密码或 Token

### 2.2 安全层面
- ❌ 禁止使用弱密码（如 `admin/123456`）作为测试账号
- ❌ 禁止在日志中记录用户密码
- ❌ 禁止在错误信息中返回堆栈跟踪或数据库错误详情
- ❌ 禁止使用自签名证书或跳过 HTTPS 验证

### 2.3 流程层面
- ❌ 禁止跳过登录验证直接访问受保护页面
- ❌ 禁止在未授权情况下调用登录接口
- ❌ 禁止使用 GET 请求提交登录表单

## 3. 无副作用检查

### 3.1 数据无副作用
- [ ] 登录失败不会创建任何用户会话或 Token
- [ ] 登录失败不会修改数据库中的用户数据
- [ ] 登录成功不会修改用户密码或其他敏感信息
- [ ] 登录操作不会触发任何非必要的数据库写入操作

### 3.2 状态无副作用
- [ ] 登录失败后，页面状态恢复到登录前（输入框内容可保留）
- [ ] 登录成功后，浏览器历史记录中不保留密码信息
- [ ] 多次登录失败不会导致账号被锁定（除非有明确的安全策略）
- [ ] 登录页面刷新后，不保留任何登录状态

### 3.3 外部系统无副作用
- [ ] 登录操作不会触发不必要的第三方 API 调用
- [ ] 登录操作不会发送不必要的邮件或短信通知
- [ ] 登录操作不会修改其他用户的会话状态
- [ ] 登录操作不会影响系统的缓存或 CDN 状态
