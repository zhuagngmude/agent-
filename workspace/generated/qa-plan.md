# 验收检查切片

## 1. 验收步骤

### 1.1 功能验收
- [ ] 界面元素完整显示（标题、按钮、输入框等）
- [ ] 所有交互功能可用（点击、输入、滚动等）
- [ ] 数据展示正确（无乱码、格式错误）
- [ ] 响应式布局正常（不同分辨率下显示合理）

### 1.2 流程验收
- [ ] 用户操作流程完整闭环
- [ ] 异常流程有合理提示
- [ ] 状态切换正确（加载中、空状态、错误状态）
- [ ] 数据持久化正常（如有）

### 1.3 性能验收
- [ ] 页面加载时间 < 3秒
- [ ] 交互响应时间 < 500ms
- [ ] 无内存泄漏（长时间操作后无明显卡顿）

## 2. 禁止路径

### 2.1 功能禁止
- ❌ 不允许出现未处理的异常崩溃
- ❌ 不允许数据丢失或错误覆盖
- ❌ 不允许界面卡死或无响应
- ❌ 不允许未授权的数据访问

### 2.2 交互禁止
- ❌ 不允许无反馈的静默失败
- ❌ 不允许重复提交导致数据重复
- ❌ 不允许输入未做合法性校验
- ❌ 不允许导航后状态丢失

### 2.3 安全禁止
- ❌ 不允许明文存储敏感信息
- ❌ 不允许跨站脚本（XSS）漏洞
- ❌ 不允许未经验证的输入直接渲染

## 3. 无副作用检查

### 3.1 数据副作用
- [ ] 读取操作不修改原始数据
- [ ] 写操作有确认机制
- [ ] 删除操作有二次确认
- [ ] 数据变更可追溯

### 3.2 界面副作用
- [ ] 弹窗/对话框不影响底层状态
- [ ] 动画不阻塞用户操作
- [ ] 滚动位置在导航后合理恢复
- [ ] 焦点管理不造成操作混乱

### 3.3 系统副作用
- [ ] 不修改浏览器/系统默认行为
- [ ] 不产生非预期的网络请求
- [ ] 不占用过多系统资源
- [ ] 关闭/卸载后无残留
