# agent蜂群 UI 原型交接说明

## 当前版本

`design/index.html` 是当前认可的第一版桌面端 Web App 静态原型。它基于 image2 参考图确定整体视觉方向，但正式页面使用 HTML 真文本，避免生图模型造成中文乱码。

## 包含页面

1. 项目总览
2. Agent 编排
3. 任务拆解与分配
4. 执行流水线详情
5. 本地 Runner 执行确认
6. Git 保存与历史记录
7. 知识库 / 文档管理
8. 系统设置

## 设计路线

- image2 用于确定视觉风格和大布局。
- HTML 原型用于真实 UI、真实中文、真实页面结构。
- 当前版本不再沿用旧的 8 套风格方向稿。

## 后续开发建议

- 将 `design/index.html` 拆分为前端组件。
- 右侧上下文面板应根据当前页面和选中对象变化。
- Runner 执行确认页需要继续强化二次确认、操作类型、Git checkpoint、影响文件和风险解释。
- Agent 编排页可继续演进为关系图 + 队列 + Agent 详情三栏结构。
- 任务拆解页需要增加状态、Agent、风险筛选，以及批量重新分配和进入审查操作。

## 安全原则

所有本地 Runner 代码执行、文件写入、删除文件、执行命令、网络调用都必须经过用户确认。高风险操作不能和普通主按钮同级展示，应有更明确的风险说明和二次确认。
