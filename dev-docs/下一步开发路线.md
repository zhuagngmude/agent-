# 下一步开发路线

日期：2026-06-11

## 当前状态

- 当前阶段：MVP-0.3。
- 已有主线：项目想法 -> `project_plan` 审批 -> 5 个 queued 任务 -> 5 条只读 Runner request 记录。
- Agent config 的安全闭环已经单独收口：apply / version history / rollback dry-run / rollback review 都保持只读或特性关闭。
- Model Gateway 仍是禁用态，真实模型调用和真实 Runner 执行都不开放。

## 当前最重要的边界

- 不做云同步。
- 不做完整权限系统。
- 不把只读 Runner 请求变成可执行命令。
- 不直接启用真实回滚写入，除非先补齐 feature flag、二次确认、Git checkpoint、rollback acceptance 和无副作用验收。

## 下一步应该做什么

1. 保持 MVP-0.3 的项目计划审批、任务拆分和只读 Runner 队列稳定。
2. 保持 Agent config 安全闭环的文档、脚本和 UI 一致。
3. 把下一阶段正式收束到 `MVP-0.4` 真实执行层计划表。
4. 继续把 Model Gateway 的禁用态、relay 说明和验证脚本分开维护。
5. 新增功能前先收敛文档，避免重复叙述边界和阶段状态。

## 仍然不要做的事

- 真实模型规划。
- 真实 Agent 触发。
- 真实 Runner 执行。
- 本地项目文件写入。
- Git mutation。
- 云同步。
- 完整权限系统落地。
