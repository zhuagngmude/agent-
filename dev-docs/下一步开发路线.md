## 老师反馈后的调整结论

本轮不再继续在旧手搓架构上叠功能。需要先重新确认项目工程方案，再决定如何初始化新工程和迁移旧原型价值。

已确认的调整方向：

- Agent 宪法、维护手册、交接说明和阶段路线默认使用中文。
- 前端不继续手搓，后续使用成熟 UI 技术栈。
- 桌面端是主入口，Web 只作为辅助预览或后续扩展入口。
- Web 和桌面端必须共用同一套 UI 源码。
- 桌面端使用 Rust/Tauri 作为宿主，Rust 负责本地能力，不重新手画 UI。
- 根目录重复前端、`apps/web` 前端、`packages/ui` 预留等目录需要重新规划，不能继续混乱叠加。
- 旧原型归档方案已确认：旧原型冻结为参考资产，不直接删除；等新架构覆盖后再讨论清理。
- 新项目目录架构已确认：`packages/ui` 为唯一 UI 源，`apps/desktop` 为主入口，`apps/web` 为辅助预览入口。
- `packages/ui` 已从预留 README 升级为 React + TypeScript + Vite + Ant Design 的最小可运行骨架。

- 不继续手搓前端。
- 不继续扩展 Node.js 原生 HTTP 后端。
