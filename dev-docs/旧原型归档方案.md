# 旧原型归档方案

日期：2026-06-13

## 一、结论

旧原型不立即删除，也不继续扩展。它的定位从“当前开发主线”调整为“冻结参考资产”。

后续正式开发以 `packages/ui`、`apps/desktop`、`data/migrations`、`data/seed`、`docs` 和 `dev-docs` 为主线。旧原型只用于迁移对照、业务规则回看和验收参考。

## 二、正式主线目录

以下目录继续作为新架构主线推进：

| 路径 | 定位 | 下一步 |
|------|------|--------|
| `packages/ui/` | 唯一共享 UI 源码 | 逐步承接旧页面的核心信息结构和交互 |
| `apps/desktop/` | 桌面端主入口，Tauri 2 + Rust 宿主 | 补齐只读 commands，后续再讨论写入 |
| `data/migrations/` | SQLite 结构真相源 | 仅通过 migration 演进表结构 |
| `data/seed/` | 新架构初始 seed 来源 | 仅保留可迁移的最小数据快照 |
| `docs/` | 正式技术、治理、验收文档 | 记录稳定规则和架构真相 |
| `dev-docs/` | 阶段路线、交接、决策过程 | 记录讨论、归档方案和阶段状态 |

## 三、冻结参考目录

以下目录和文件保留，但不再作为正式工程主线继续扩展：

| 路径 | 冻结原因 | 可参考内容 | 禁止事项 |
|------|----------|------------|----------|
| `apps/web/` | 旧 Web 原型，原生 HTML / CSS / JavaScript | 页面信息结构、交互流程、Mock UI 状态 | 不继续新增正式页面，不继续做新业务功能 |
| `services/api/` | 旧 Node.js 原生 HTTP Mock API | 状态流转、旧 REST endpoint、Mock 数据生成逻辑 | 不继续扩展为正式后端，不新增正式路由 |
| `design/index.html`、`design/script.js`、`design/styles.css` | 设计原型和视觉参考 | 布局、色彩、早期展示意图 | 不作为正式 UI 源码 |
| `data/mock/` | 旧 Mock 运行态与验证数据 | 旧状态机和回归参考 | 不手动改 `runtime-state.json`，不作为新运行库 |
| 旧 Mock / SQLite 验证脚本 | 旧链路回归资产 | 对照旧 MVP 是否被破坏 | 不把旧脚本当成新架构唯一验收标准 |

## 四、暂不处理目录

以下目录当前不删除、不迁移、不扩展，只保留占位或未来设计讨论入口：

| 路径 | 当前处理 |
|------|----------|
| `services/runner/` | Runner 真实执行仍关闭，暂不实现 |
| `services/worker/` | 暂无新架构职责，保留不动 |
| `packages/agent-core/` | 后续承接业务模型和编排规则，当前不提前填充 |
| `packages/shared/` | 后续承接共享类型和工具，当前不提前填充 |

## 五、迁移原则

1. **先冻结，再迁移，最后清理**：没有新架构等价能力前，不删除旧原型。
2. **迁移的是业务价值，不迁移旧实现方式**：旧页面的流程和信息结构可以参考，但不复制原生 HTML / CSS / JavaScript。
3. **新 UI 只写在 `packages/ui/`**：桌面端和 Web 预览都消费同一套 UI 源码。
4. **新后端只走 Tauri commands**：不继续把 `services/api/` 扩展成正式后端。
5. **只读先行**：先完成 `projects / agents / tasks / approvals` 的只读链路，再讨论写入和审批流。
6. **保护路径不动**：`design/image2/`、`data/mock/runtime-state.json`、`data/local/`、`logs/`、`.playwright-cli/`、`_internal/` 仍按宪法保护。

## 六、未来清理条件

满足以下条件后，才讨论删除或移动旧目录：

1. `packages/ui` 已覆盖旧 `apps/web/` 的核心页面信息结构。
2. `apps/desktop` 已覆盖旧 `services/api/` 的核心只读数据能力。
3. 旧 Mock 状态流转已经在新架构文档、SQLite seed 或 Tauri commands 中有等价表达。
4. 对应验收步骤已经更新到 `docs/demo-checklist.md` 或新验收文档。
5. 删除或迁移前已创建 Git 存档，并列出可回滚方式。

## 七、当前下一步

1. 更新阶段路线：第 7 步“旧原型归档方案确认”标为已完成。
2. 进入只读 commands 小步：`list_agents`、`list_tasks`、`list_approvals`。
3. 只读 commands 验证通过后，再让 `packages/ui` 从 SQLite 读取真实只读数据。
