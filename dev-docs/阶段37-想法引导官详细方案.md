# 阶段 37 想法引导官详细方案

> 更新提示（2026-06-16）：阶段 37 已完成。本文件末尾提到的旧阶段 38“自动工作流模板”已顺延；当前下一步是阶段 38“项目类型分流与通用想法入口”，详见 `dev-docs/当前项目导航.md`。

## 1. 背景

阶段 36 已完成“全自动模式宪法与权限边界”：

- 默认模式仍是 `manual`。
- `full_auto` 必须用户显式开启。
- 总控智能体可按策略自动授权 L0-L3。
- 阶段 36 不开放自动 L4 执行。
- 所有模型调用、授权、执行准备和后续执行都必须审计。

阶段 37 进入全自动路线的第一块产品能力：**想法引导官**。

用户现在的目标不是只生成一份普通项目计划，而是让系统主动帮助用户把粗想法扩宽、追问、整理成可执行项目种子。想法引导官是全自动 Agent 蜂群的入口角色，但本阶段仍不开放自动执行。

本阶段只做：

```text
用户粗想法 -> 想法引导官追问/扩展 -> 项目种子草案
```

不做：

```text
项目种子 -> 自动拆任务 -> 自动 Runner -> 写文件 -> 自动交付
```

## 3. 本阶段目标

阶段 37 的目标：

1. 新增“想法引导官”角色定义。
2. 定义想法引导输入、追问、扩展和输出结构。
3. 定义项目种子草案数据形态。
4. 定义想法引导官的权限级别：L1 模型草案能力，不执行 Runner。
5. 定义真实模型调用边界：必须走 Model Gateway、模型目录和 model_calls 脱敏审计。
6. 定义前端入口：在项目计划页或新工作台入口中提供“扩展想法”流程。
7. 定义保存方式：输出可保存为项目种子草案，后续再转项目计划草案。
8. 同步治理文档和路线图。

## 4. 非目标

本阶段禁止扩大范围：

- 不开放自动 Runner。
- 不执行命令。
- 不写用户项目文件。
- 不改 Git。
- 不创建 tasks。
- 不创建 runner_requests。
- 不自动审批项目计划。
- 不创建 preflight/gate/dry-run/lock/minimal_run。
- 不新增前端自由 provider、key、base URL、prompt、header、body。
- 不把 Skill 原文自由拼进 provider 请求。
- 不让模型直接决定最终任务结构。
- 不提交 Git。

## 5. 权限边界

想法引导官属于阶段 36 权限体系中的：

```text
L1：模型草案
```

允许：

- 读取用户输入的粗想法。
- 读取用户填写的约束。
- 通过 Model Gateway 调用受控模型。
- 生成追问列表。
- 生成项目种子草案。
- 写入 `model_calls` 安全审计。
- 将脱敏、限长后的项目种子保存到本地 SQLite。

禁止：

- 执行 Runner。
- 写用户项目文件。
- 改 Git。
- 创建任务。
- 创建审批。
- 创建 Runner 请求。
- 绕过 Model Gateway。
- 保存 raw prompt。
- 保存 raw response。
- 保存 raw provider error。
- 读取或展示 raw key/base URL。

## 6. 核心流程

第一版建议分两步，不要一步到位全自动。

### 6.1 生成追问

用户输入：

```text
粗想法
约束（可选）
目标平台（可选）
预算/时间（可选）
```

想法引导官输出：

```text
clarifying_questions: 3-7 个问题
assumptions: 系统当前假设
risk_notes: 已发现风险
suggested_direction: 推荐方向
```

要求：

- 问题必须是中文。
- 问题要帮助用户扩宽想法，不是机械收集字段。
- 不要问密钥、账号、隐私、支付凭据等敏感信息。
- 不要要求用户上传源码或文件。
- 不要暗示已经开始执行。

### 6.2 生成项目种子

用户补充回答后，想法引导官输出项目种子：

```text
product_goal
target_users
core_scenarios
mvp_scope
non_goals
key_features
pages_or_modules
data_entities
external_integrations
technical_constraints
acceptance_criteria
risk_points
open_questions
recommended_next_step
```

要求：

- 输出必须是结构化 JSON 或后端结构体可解析形态。
- 用户可见显示为中文。
- 不能包含 raw prompt/raw response。
- 不能包含密钥、Token、账号密码。
- 如果模型输出含敏感模式，必须拒绝保存或进行脱敏。

## 7. 数据模型建议

阶段 37 建议新增 migration，但第一版必须保持安全保守。

```text
data/migrations/013_add_idea_guidance.sql
```

### 7.1 idea_guidance_sessions

用途：记录一次想法引导会话。

建议字段：

```sql
CREATE TABLE IF NOT EXISTS idea_guidance_sessions (
  id TEXT PRIMARY KEY,
  project_id TEXT NOT NULL,
  status TEXT NOT NULL CHECK (status IN ('draft', 'questions_ready', 'seed_ready', 'cancelled', 'failed')),
  source TEXT NOT NULL CHECK (source IN ('manual', 'model_guided')),
  idea_summary TEXT NOT NULL,
  constraints_summary TEXT,
  model_call_id TEXT,
  created_by TEXT NOT NULL,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  FOREIGN KEY (project_id) REFERENCES projects(id),
  FOREIGN KEY (model_call_id) REFERENCES model_calls(id)
);
```

安全要求：

- `idea_summary` 是用户粗想法的限长摘要，不保存超长原文。
- 如果保存原始用户输入，需要明确字段、长度和敏感值拦截；第一版建议不保存 raw idea。
- `model_call_id` 只关联安全审计记录。

### 7.2 idea_guidance_questions

用途：保存模型生成的追问。

```sql
CREATE TABLE IF NOT EXISTS idea_guidance_questions (
  id TEXT PRIMARY KEY,
  project_id TEXT NOT NULL,
  session_id TEXT NOT NULL,
  sort_order INTEGER NOT NULL,
  question TEXT NOT NULL,
  answer TEXT,
  status TEXT NOT NULL CHECK (status IN ('pending', 'answered', 'skipped')),
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  FOREIGN KEY (project_id) REFERENCES projects(id),
  FOREIGN KEY (session_id) REFERENCES idea_guidance_sessions(id)
);
```

安全要求：

- `question` 必须中文、限长、脱敏。
- `answer` 用户输入必须限长、敏感值拦截。
- 不允许问题要求用户输入 key/token/password。

### 7.3 project_seeds

用途：保存项目种子草案。

```sql
CREATE TABLE IF NOT EXISTS project_seeds (
  id TEXT PRIMARY KEY,
  project_id TEXT NOT NULL,
  session_id TEXT NOT NULL,
  status TEXT NOT NULL CHECK (status IN ('draft', 'ready', 'converted', 'cancelled')),
  product_goal TEXT NOT NULL,
  target_users TEXT NOT NULL,
  mvp_scope TEXT NOT NULL,
  non_goals TEXT NOT NULL,
  key_features TEXT NOT NULL,
  pages_or_modules TEXT NOT NULL,
  data_entities TEXT,
  technical_constraints TEXT,
  acceptance_criteria TEXT NOT NULL,
  risk_points TEXT,
  open_questions TEXT,
  recommended_next_step TEXT NOT NULL,
  model_call_id TEXT,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  FOREIGN KEY (project_id) REFERENCES projects(id),
  FOREIGN KEY (session_id) REFERENCES idea_guidance_sessions(id),
  FOREIGN KEY (model_call_id) REFERENCES model_calls(id)
);
```

说明：

- 多值字段第一版可存 JSON string，但必须严格解析。
- 不要存模型 raw response。
- 后续阶段可以把 `project_seeds` 转成 `project_plan_drafts`，但阶段 37 不自动创建任务或 Runner 请求。

## 12. 安全和脱敏要求

必须拦截：

```text
sk-
api_key
token=
password=
Authorization: Bearer
-----BEGIN
http://
https://
```

说明：

- 用户想法中出现普通网址可能是产品需求，但第一版可以保守拒绝或脱敏。
- 如果后续要允许 URL，必须单独设计 URL 安全策略，不在阶段 37 放开。

输出限制：

- 单个问题建议不超过 200 字。
- 问题数量 3-7 个。
- 项目种子每个字段限长。
- 总摘要限长。
- 模型响应过大返回 `response_too_large`。

## 13. 状态机

### 13.1 idea_guidance_sessions

```text
draft
  -> questions_ready
  -> seed_ready
  -> cancelled
  -> failed
```

规则：

- `draft`：用户刚输入想法，未调用模型。
- `questions_ready`：追问已生成。
- `seed_ready`：项目种子已生成。
- `cancelled`：用户取消。
- `failed`：模型调用或审计失败。

### 13.2 project_seeds

```text
draft
  -> ready
  -> converted
  -> cancelled
```

规则：

- `ready` 才能转项目计划草案。
- `converted` 后不能重复转换。
- 转换必须幂等。

2. 缺少二次确认时拒绝。

4. 前端传自由 prompt/header/body 时被 `deny_unknown_fields` 拒绝。
5. 模型输出空内容拒绝。
6. 模型输出非 JSON 拒绝。
7. 模型输出字段缺失拒绝。
8. 模型输出含敏感值拒绝或脱敏。
9. 审计写入失败时不返回模型结果。
10. 生成追问不创建 tasks/runner_requests/runtime_events。
11. 生成项目种子不执行 Runner、不写文件、不改 Git。
12. 跨项目隔离。
13. 幂等保存。
14. project_seed 转草案不重复创建审批。

前端验证：

1. 浏览器预览模式按钮不可用。
2. 桌面模式显示二次确认。
3. 追问列表中文展示。
4. 项目种子预览不显示 raw JSON 大块乱码。
