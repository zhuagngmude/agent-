# 阶段 35.3 桌面端主控台体验收口详细方案

## 1. 背景

当前 `agent-swarm` 已完成阶段 0-35.1：

- 阶段 25/35：真实模型调用和受控模型目录已开放。
- 阶段 34：极小范围真实 Runner 已开放。
- 阶段 35.1：前端中文化已做过一轮收口。

但桌面端主控台仍有明显工程骨架残留：

- 顶部仍有“共享 UI 工程骨架”等开发期文案。
- 浏览器预览和 Tauri 桌面真实模式区分不够清楚。
- fallback 数据伪装成真实 running / pending 状态。
- 小窗口布局容易拥挤。
- 部分 UI 文件存在乱码文案残留，需要一并修正。

本阶段目标不是做漂亮网站，而是把桌面端收口成“稳、清楚、可控”的本地 AI 协作控制台。

## 2. 本阶段目标

让用户一打开应用就能看懂：

1. 当前运行在浏览器预览，还是桌面真实模式。
2. 当前数据是真实 SQLite 数据，还是只读示例数据。
3. 当前 Runner 和真实模型能力处于什么边界。
4. 窄窗口下界面不乱、不竖排、不遮挡。

## 3. 非目标

本阶段禁止扩大范围：

- 不改 Rust / Tauri command。
- 不改数据库 migration。
- 不改模型调用逻辑。
- 不改 Runner 权限。
- 不改 `ProjectPlanPage` 的业务流程结构；步骤式工作台放到阶段 35.4。
- 不新增依赖。
- 不提交 Git。
- 不删除旧文档或旧原型。

## 4. 建议修改文件

只允许优先修改以下文件：

```text
packages/ui/src/layouts/AppShell.tsx
packages/ui/src/theme/global.css
packages/ui/src/pages/OverviewPage.tsx
packages/ui/src/pages/SettingsPage.tsx
packages/ui/src/utils/desktopHost.ts
```

如确实需要同步导航文案，可小范围修改：

```text
packages/ui/src/routes/mainNavItems.ts
```

## 5. 必须修正的现状

### 5.1 AppShell 文案

把开发期文案改成产品态文案。

建议：

- 品牌：`agent蜂群`
- 顶部标题：`agent蜂群控制台`
- 副标题：`本地 AI 协作工作台`

删除或替换：

- `共享 UI 工程骨架`
- `Tauri 预留`
- `Runner 关闭`
- `后续迁移`
- 其他“工程骨架 / 预留 / fallback”这类直接暴露开发过程的文案

### 5.2 运行模式提示

必须让用户一眼看懂当前模式。

浏览器预览模式：

```text
浏览器预览 · 只读示例数据
```

桌面真实模式：

```text
桌面模式 · 本地 SQLite
```

连接失败但显示示例数据：

```text
桌面连接失败 · 已切换只读示例数据
```

要求：

- 该模式提示应出现在主控台顶部或 Overview 明显位置。
- 不要把浏览器 fallback 数据说成真实数据。
- 所有写入按钮在浏览器模式仍必须不可用。

### 5.3 fallback 数据

`packages/ui/src/utils/desktopHost.ts` 中 fallback 数据必须明确是示例。

建议命名：

- 项目名：`示例项目：agent蜂群预览`
- Agent：
  - `示例 Agent：产品规划`
  - `示例 Agent：前端实现`
- 任务：
  - `示例任务：整理项目想法`
  - `示例任务：生成前端预览`
- 审批：
  - `示例审批：只读预览确认`

要求：

- 不要使用容易被误认为真实运行的状态组合。
- 不要用 `running` / `pending` 伪装真实进行中流程。
- 如果类型必须使用合法状态，可在标题和说明里明确带 `示例`。
- `reason`、`description` 等文案必须写清楚“这是浏览器预览示例数据”。

### 5.4 OverviewPage

总览页要从“工程骨架介绍”改为“控制台概览”。

建议结构：

- 标题：`控制台总览`
- 副标题：`查看本地项目、Agent、任务和审批状态`
- 运行模式卡片：
  - 浏览器预览 / 桌面模式 / 连接失败
- 能力边界卡片：
  - 真实模型：`仅通过受控模型目录和 Model Gateway 人工触发`
  - Runner：`阶段 34 仅允许沙箱路径、固定 Git 只读命令和人工二次确认`
  - 数据：`桌面模式读取本地 SQLite；浏览器模式只显示示例数据`

### 5.5 SettingsPage

设置页不能再写旧口径：

```text
真实 provider、raw key、Model Gateway 实现当前不开放
```

应改为：

```text
真实模型已通过 Model Gateway 和受控模型目录开放；前端不接收 key、base URL、prompt、header 或 provider body。
```

并补充：

```text
Runner 仅在阶段 34 边界内开放：临时沙箱路径、固定 Git 只读命令、人工二次确认。
```

### 5.6 中文和乱码

本阶段必须清理目标文件里的乱码文案。

要求：

- 用户可见文案必须是正常中文。
- 不得出现乱码字符。
- 后端枚举、数据库字段名、内部状态不得直接裸露给用户。
- 英文只保留在模型 ID、命令、环境变量、Provider、路径、数据库 ID 等技术标识中。

建议扫描：

```powershell
rg -n "�|铚|鍦|涓|妗|瀹|绯|棰|fallback|工程骨架|后续迁移|Tauri 预留|Runner 关闭" packages/ui/src/layouts packages/ui/src/pages packages/ui/src/utils
```

说明：

- `fallback` 可以作为代码变量保留。
- 用户可见文案里不要出现 `fallback`。

## 6. 响应式布局要求

目标：窄窗口可用，不追求手机 App。

要求：

- 小于约 `760px` 时，侧边栏不得挤压内容导致文字竖排。
- 可以改成顶部导航、横向滚动导航或折叠导航。
- 顶部 Header 内容允许换行，但不能遮挡主体内容。
- 导航按钮文字不能被压成单字竖排。
- 页面主体 `min-width: 0`，表格和卡片不能撑爆外壳。

建议：

- `.app-shell` 在窄窗口使用纵向布局。
- `.app-shell__sider` 变成顶部区域。
- `.app-shell__nav` 改为横向滚动或 wrap。
- `.app-shell__nav-item` 设置稳定高度和 `white-space: nowrap`。

## 7. 文案统一口径

真实模型：

```text
真实模型仅允许通过受控模型目录和 Model Gateway 人工触发。
前端不接收 key、base URL、prompt、header 或 provider body。
```

Runner：

```text
阶段 34 仅允许沙箱路径、固定 Git 只读命令和人工二次确认。
当前步骤不会执行 Runner。
```

浏览器模式：

```text
浏览器预览只显示只读示例数据，写入和真实调用不可用。
```

桌面模式：

```text
桌面模式读取本地 SQLite，并通过 Tauri commands 执行受控操作。
```

## 8. 验收标准

必须满足：

1. 窄窗口不再文字竖排。
2. 用户一眼知道自己在“浏览器预览”还是“桌面真实模式”。
3. fallback 数据明确是“示例数据”。
4. 页面无“共享 UI 工程骨架”“后续迁移”“Tauri 预留”“Runner 关闭”等开发期文案。
5. Runner 和真实模型文案不冲突。
6. 目标文件无用户可见乱码。
7. 控制台没有 Ant Design warning。
8. 不改 Rust / TS 业务链路，只做前端外壳和文案收口。
