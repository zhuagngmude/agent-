# 后端骨架搭建执行清单

日期：2026-06-13

本文只是一份执行清单，不重复写架构规范。架构与边界以 [docs/backend-design.md](../docs/backend-design.md) 和 [docs/backend-architecture.md](../docs/backend-architecture.md) 为准。

## 执行顺序

### 1. 检查 Rust 环境

先确认本机可用：

- `rustc`
- `cargo`
- `cargo tauri`

检查不通过就先补环境，不进入骨架搭建。

### 2. 搭最小后端骨架

目标只做最小可启动闭环：

- `apps/desktop/src-tauri/` 能作为 Tauri 宿主入口
- 接上 `packages/ui/` 作为前端 UI
- 先放 1 到 2 个最小 command 验证 `invoke`

第一版不要求接业务数据库写入。

### 3. 验证最小闭环

验证项按顺序确认：

- `cargo tauri dev` 能启动
- 前端能调用 Rust command
- 启动过程无明显报错
- 没有越过当前边界去做真实 Runner、真实模型或云同步

## 执行原则

- 按顺序执行，不跳步。
- 每一步完成后再进入下一步。
- 遇到缺依赖或缺工具，先记录，再补齐。
