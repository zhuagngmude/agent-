# 项目清理决策清单

日期：2026-06-19

用途：整理项目文件时使用。目标是让项目变干净，同时避免误删历史记录、用户产物或仍被当前主链路引用的代码。

## 可以直接整理

- 入口文档里的过期状态：`README.md`、`dev-docs/README.md`、`dev-docs/当前项目导航.md`、`dev-docs/下一步开发路线.md`、`dev-docs/新窗口交接说明.md`、`dev-docs/AI开发维护手册.md`。
- 明显误导当前主线的表述，例如“P0 仍待实现”“Runner 主链路尚未接入”“任务页尚未展示 Agent 信息”。
- 没有真实功能的主导航入口、假按钮、旧提示文案。清理前先用 `rg` 确认没有当前页面或 Tauri command 依赖。
- 文档里的旧计划入口：保留链接可以，但必须标注“阶段背景”或“历史记录”。

## 先保留为历史

- `dev-docs/阶段*.md`：阶段设计记录，不逐篇重写，也不批量删除。
- `docs/runner-safety-acceptance.md`、`docs/tauri-readonly-skeleton-acceptance.md`、`docs/agent-permission-contract.md`、`docs/agent-config-apply-dry-run-spec.md`：旧阶段验收或契约文档，里面的“只读”“不执行”“feature_disabled”不等于当前主线限制。
- `services/api/`、`apps/web/`：旧 Node.js / Web 原型路径，当前不是主线，但删除前必须确认没有文档、脚本或测试仍引用。
- `runner_minimal_run.rs` 中因 `#[cfg(test)]` / `#[cfg(not(test))]` 产生的 unused 警告：不要因为警告直接删函数。

## 需要用户确认后再处理

- `workspace/generated/` 下的项目产物、删除记录或历史生成结果。
- 任何真实文件删除、目录清空、批量移动。
- Git commit、push、reset、checkout、rebase。
- 保护路径写入或删除。
- 旧 Web/API 原型的大范围移除。

## 不要碰

- `design/image2/`
- `data/local/`
- `logs/`
- `_internal/`
- `data/mock/runtime-state.json`
- 任何 API Key、Token、私钥、`.env` 内容或登录凭证。
