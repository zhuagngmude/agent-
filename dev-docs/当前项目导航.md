# 当前项目导航

日期：2026-06-17

这份文档是给人和下一位智能体看的“先读入口”。历史阶段文档保留为记录，不再要求每次全部读完。

## 一句话方向

`agent-swarm` 是一个本地多模型 Agent 调度系统：用户输入项目目标，总控 Agent 自动判断项目类型、选择固定员工和项目专家，按职责边界拆任务，通过 Codex/Claude/DeepSeek/Gemini/Cursor/OpenCode 等模型或执行器协作，最后经 Runner、审批和运行记录安全产出项目文件。

当前第一目标不是继续堆页面，而是把这条链路稳定成真的可用产品：

```text
主控台输入目标
-> 总控判断项目类型、技术栈、风险和阶段
-> 选择固定员工池和项目专家 Agent
-> 按职责边界拆任务和转派
-> 通过模型网关调用对应模型或执行器
-> Runner / 审批 / 运行记录
-> workspace/generated 查看结果
```

## 当前事实

- 桌面端是主入口，Web 只作为辅助预览或历史参考。
- 当前主导航：主控台、任务拆解、流程蓝图、运行输出、AI 员工、审批与安全、系统设置。
- AI 员工页已经开始承载全技术栈固定员工池、项目专家推荐、职责边界和执行器/模型选择。
- 任务页应按“总任务”组织，不再把一堆子任务平铺到左侧。
- 已完成项目的文件夹位置是 `F:\Projects\agent-swarm\workspace\generated`。
- 模型配置在系统设置中完成：API Key、Base URL、模型 ID 保存到本机系统凭据，并在桌面端启动后恢复到当前进程环境变量；输入框保存后清空，不落库、不进日志。
- 主控台“问总控”在桌面端会调用当前模型服务返回对话；浏览器预览只保留本地解释 fallback。
- Runner 全自动主链路允许受控执行。用户已经明确要求“全自动”和“Runner 全面开放”。
- 仍禁止自由 shell、Git commit/push、文件删除、保护路径写入、提交密钥、把 raw key/raw prompt/raw response 写入日志或数据库。
