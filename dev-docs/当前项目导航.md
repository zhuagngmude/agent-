# 当前项目导航

日期：2026-06-16

这份文档是给人和低级智能体看的“先读入口”。历史阶段文档保留为记录，不再要求每次全部读完。

## 一句话方向

agent蜂群是一个本地 AI 项目总控台：用户只提出想法和目标，系统负责引导想法、识别项目类型、生成项目种子、拆解任务、分配智能体、审查风险，并在受控边界内逐步执行。

当前主线不是继续堆模块，而是打通：

```text
一句想法
-> 项目类型分流
-> 想法引导
-> 项目种子
-> 项目计划
-> 智能体分工
-> 审查 / 许可 / 预演 / 锁定
-> 极小范围执行
-> 复盘与下一轮
```

## 当前事实边界

- 阶段 34 已开放极小范围真实执行：仅临时沙箱路径、固定版本管理只读命令、人工二次确认；不开放批量/自动执行、自由命令、版本提交/推送、文件删除、网络请求。
- 阶段 35 已开放受控模型目录：前端只能选择后端目录里启用的模型；不开放前端自由 key、base URL、prompt、header、body。
- 阶段 35.1-35.4 已完成前端中文化、桌面主控台和项目计划页步骤式工作台第一版。
- 阶段 36 已完成全自动模式宪法与权限边界方案；不代表自动执行已开放。
- 阶段 37 已完成想法引导官能力：可通过受控模型调用生成引导问题和项目种子，仍不执行、不写文件、不改版本。

## 下一阶段

### 阶段 38：项目类型分流与通用想法入口

目标：把首页变成真正的第一入口。用户只输入一句“我想做什么”，系统先判断项目类型，再进入对应想法引导模板。

第一版项目类型：

1. 软件产品
2. AI 自动化
3. 内容创作
4. 商业方案
5. 通用目标

阶段 38 只做入口、分类、推荐问题和记录，不做真实执行扩权。

允许做：

- 首页主控输入框真实可用。
- 规则分类项目类型。
- 保存 intake session。
- 展示识别类型、判断理由、推荐问题。
- 进入现有想法引导官或项目计划链路。

禁止做：

- 不调用自由模型。
- 不创建任务。
- 不创建审批。
- 不执行执行引擎。
- 不写用户项目文件。
- 不改版本。
- 不接前端 key/base URL/prompt 自由配置。

## 阶段顺序修正

旧的“阶段 38 多 Provider Adapter”先顺延，不是当前下一步。原因是产品主路径还不够清楚，先做项目类型分流，才能知道不同模型、角色和技能应该服务什么场景。

建议新顺序：

```text
38 项目类型分流与通用想法入口
39 多 Provider Adapter
40 角色模型和 Skill 绑定
41 全自动主控智能体与授权流
42 对话式主控台接管旧流程
43 全项目大体检
```

## 历史文档处理原则

- 历史阶段文档不删除，它们记录当时边界。
- 如果历史文档里写“当时不开放”，不一定代表当前仍不开放，以本导航、宪法、维护手册和交接说明为准。
- 新开发方案应优先写新阶段文档，不要在旧阶段文档里反复覆盖历史。
- 文档冲突时，以 `docs/Agent宪法.md`、`docs/AI开发细则.md`、`dev-docs/当前项目导航.md` 为优先。
