# 阶段 16：独立页面拆分方案

日期：2026-06-14

本文是阶段 16 的设计文档，定义如何把 `OverviewPage` 中的内容拆分为独立页面，并扩展侧边栏导航。

## 一、目标

把当前集中在 `OverviewPage` 的所有内容拆开：

| 页面 | 职责 | 数据源 | 写入能力 |
|------|------|--------|----------|
| `OverviewPage` | 总览仪表盘：统计卡片 + 连接状态 | `useDesktopHostOverview` | 无 |
| `TasksPage` | 任务列表 + 新建 + 状态操作 | tasks + agents | `create_task`、`update_task_status` |
| `AgentsPage` | Agent 列表（第一版只读） | agents | 无 |
| `ApprovalsPage` | 审批列表 + 通过/拒绝/仅补丁 | approvals | `approve_approval`、`reject_approval`、`patch_only_approval` |
| `SettingsPage` | 占位页：项目信息只读展示 | project | 无 |

## 二、页面导航

### 2.1 不使用 React Router

第一版用 `useState` + `activePage` 实现页面切换：

```ts
type PageKey = "overview" | "tasks" | "agents" | "approvals" | "settings";
const [activePage, setActivePage] = useState<PageKey>("overview");
```

原因：
- 项目只有 5 个页面，不需要路由库
- 避免新增依赖（React Router 会引入 `react-router-dom`）
- 桌面宿主不需要 URL 路由

后续页面增多（> 8 个）或有深层嵌套时再评估 React Router。

### 2.2 侧边栏更新

`mainNavItems.ts` 当前有 7 个导航项（含 `git`、`knowledge`），但只有 5 个有对应页面。

```ts
// 保留的导航项（有对应页面）
{ key: "overview",   label: "项目总览",   icon: LayoutDashboard }
{ key: "tasks",      label: "任务拆解",   icon: ListChecks }
{ key: "agents",     label: "Agent 编排", icon: Bot }
{ key: "approvals",  label: "审批确认", icon: ClipboardCheck }
{ key: "settings",   label: "系统设置",   icon: Settings }
```

`git` 和 `knowledge` 导航项移除（对应模块暂不做，见迁移清单）。

`mainNavItems` 不再硬编码 `active: true`，改为由 AppShell 根据 `activePage` 动态判断。

## 三、数据共享

### 3.1 选择：提升到 App.tsx

`useDesktopHostOverview()` 从 `OverviewPage` 提升到 `App.tsx`，数据通过 props 传递给各子页面。

```
App.tsx
  └── useDesktopHostOverview()  ← 只查一次
  └── AppShell
        ├── OverviewPage  ← props: project, tasks, agents, approvals
        ├── TasksPage     ← props: tasks, agents, refresh
        ├── AgentsPage    ← props: agents
        ├── ApprovalsPage ← props: approvals, refresh
        └── SettingsPage  ← props: project
```

选择理由：
- 避免每个页面独立调用 hook（5 页 × 4 invoke = 20 次调用）
- 页面切换时数据保持一致，不需要重新加载
- `refresh` 统一管理，任一页面写入后刷新全部数据

### 3.2 各页面 Props 定义

```ts
// 共享基础类型
type PageData = DesktopHostOverviewData & { refresh: () => void };

// OverviewPage — 只读仪表盘
type OverviewPageProps = PageData;

// TasksPage — 任务 CRUD
type TasksPageProps = {
  tasks: TaskSummary[];
  agents: AgentSummary[];
  refresh: () => void;
  canWrite: boolean; // 只有 status === "connected" 时为 true，控制写入按钮显隐
};

// AgentsPage — 只读列表
type AgentsPageProps = {
  agents: AgentSummary[];
};

// ApprovalsPage — 审批操作
type ApprovalsPageProps = {
  approvals: ApprovalSummary[];
  refresh: () => void;
  canWrite: boolean; // 只有 status === "connected" 时为 true，控制审批按钮显隐
};

// SettingsPage — 只读占位
type SettingsPageProps = {
  project: ProjectSummary;
};
```

## 四、组件拆分计划

### 4.1 AppShell.tsx

新增 props：

```ts
type AppShellProps = {
  activePage: PageKey;
  onNavigate: (page: PageKey) => void;
  children: ReactNode;
};
```

侧边栏按钮点击时调用 `onNavigate(key)`，`active` 状态由 `activePage === item.key` 决定。

Header 区域保留只读标签和占位图标，不触发任何写入能力：
  - `Tauri 预留` / `Runner 关闭` 标签：只读状态展示
  - Git 保存点按钮：只读占位，不触发 Git 操作（当前 Git 全局禁用）
  - 通知按钮：只读占位
  - 设置按钮：只读占位（设置入口在侧边栏）

### 4.2 App.tsx

从单纯的布局包裹变为页面编排器：

```tsx
export function App() {
  const [activePage, setActivePage] = useState<PageKey>("overview");
  const overviewState = useDesktopHostOverview();

  const renderPage = () => {
    if (overviewState.status === "loading") {
      // 用 Ant Design Spin，不新建自定义 loading 组件
      return <Spin size="large" style={{ display: "block", marginTop: 120 }} />;
    }

    // browser / connected / error 都有完整数据（error 有 fallback）
    const { refresh, ...data } = overviewState;
    const canWrite = overviewState.status === "connected";
    // 只有 Tauri 连接成功时才能写入；browser / error 模式下即使有 fallback 数据也不显示写入按钮

    switch (activePage) {
      case "overview":
        return <OverviewPage {...data} connectionStatus={overviewState.status} />;
      case "tasks":
        return <TasksPage tasks={data.tasks} agents={data.agents} refresh={refresh} canWrite={canWrite} />;
      case "agents":
        return <AgentsPage agents={data.agents} />;
      case "approvals":
        return <ApprovalsPage approvals={data.approvals} refresh={refresh} canWrite={canWrite} />;
      case "settings":
        return <SettingsPage project={data.project} />;
    }
  };

  return (
    <AppShell activePage={activePage} onNavigate={setActivePage}>
      {renderPage()}
    </AppShell>
  );
}
```

### 4.3 OverviewPage

**拆走的内容：**
- 任务表格 + 操作列 → `TasksPage`
- 新建任务 Modal → `TasksPage`
- Agent 表格 → `AgentsPage`
- 审批表格 + 操作列 → `ApprovalsPage`

**保留的内容：**
- 3 个统计卡片（任务数 / Agent 数 / 待审数）
- 桌面宿主连接状态卡片（含 loading / connected / browser / error 四种状态）

**简化后的结构：**

```tsx
export function OverviewPage({ project, tasks, agents, approvals, connectionStatus }: Props) {
  return (
    <Space>
      <PageHeading />
      <Row> {/* 3 个 Statistic 卡片 */} </Row>
      <ConnectionCard status={connectionStatus} project={project} />
    </Space>
  );
}
```

OverviewPage 不再导入 `CreateTaskModal`、`updateTaskStatus`、`approveApproval` 等写入函数。

### 4.4 TasksPage（新建）

```tsx
// 从 OverviewPage 拆出
export function TasksPage({ tasks, agents, refresh }: TasksPageProps) {
  const [createModalOpen, setCreateModalOpen] = useState(false);

  return (
    <>
      <Card title="任务拆解" extra={<Button onClick={() => setCreateModalOpen(true)}>+ 新建任务</Button>}>
        <Table columns={taskColumns + actionColumn} dataSource={taskRows} />
      </Card>
      <CreateTaskModal open={createModalOpen} agents={agents} onClose={...} onCreated={...} />
    </>
  );
}
```

任务表格、状态操作列、状态转换映射 (`statusTransitions`) 从 OverviewPage 搬过来。
写入按钮（新建任务、状态操作）只在 `canWrite === true` 时显示；否则 disabled + tooltip。

### 4.5 AgentsPage（新建）

```tsx
export function AgentsPage({ agents }: AgentsPageProps) {
  return (
    <Card title="Agent 编排">
      <Table columns={agentColumns} dataSource={agentRows} />
    </Card>
  );
}
```

纯只读，无操作列。

### 4.6 ApprovalsPage（新建）

```tsx
export function ApprovalsPage({ approvals, refresh }: ApprovalsPageProps) {
  return (
    <Card title="审批确认">
      <Table columns={approvalColumns + actionColumn} dataSource={approvalRows} />
    </Card>
  );
}
```

审批操作列（通过/拒绝/仅补丁）从 OverviewPage 搬过来。
审批操作按钮只在 `canWrite === true` 且 `status === "pending"` 时显示。

注意：阶段 16 只迁 SQLite 审批记录和终态操作，不迁 Runner diff、Runner job、文件审批详情。

### 4.7 SettingsPage（新建）

```tsx
export function SettingsPage({ project }: SettingsPageProps) {
  return (
    <Card title="系统设置">
      <Descriptions>
        <Descriptions.Item label="项目名称">{project.name}</Descriptions.Item>
        <Descriptions.Item label="项目状态">{project.status}</Descriptions.Item>
        <Descriptions.Item label="当前阶段">{project.phase}</Descriptions.Item>
```

只读占位。不展示任何 raw key、provider 配置或模型设置。
明确用 Alert 说明当前阶段不开放。

## 五、OverviewPage 加载状态处理

当前 `useDesktopHostOverview()` 返回 4 种状态：`loading` | `browser` | `connected` | `error`。

拆分后，loading 状态在 `App.tsx` 的 `renderPage()` 中统一处理：
- `loading`：所有页面显示加载骨架
- `browser` / `connected`：正常渲染数据
- `error`：正常渲染（有 fallback 数据），OverviewPage 额外展示错误信息和 Alert

## 八、边界约束

拆分过程中不引入新依赖、新后端能力、新数据表。具体禁止：

- 不安装 `react-router-dom` 或其他路由库
- 不新增 Tauri command
- 不修改 Rust 代码
- 不修改 SQLite schema
- 不在 SettingsPage 中暴露 raw key、provider 配置
- 不手搓 UI，全部使用 Ant Design 组件
- 不新增 CSS 硬编码样式
