# 重新立项说明

日期：2026-06-13

## 一、重新立项结论

当前项目不再沿用“原生 HTML / CSS / JavaScript + Node.js 原生 HTTP”的手搓扩展路线继续往前写业务代码，而是按单人、本地、自用桌面工具重新立项。

项目主形态定为：

- 单人自用
- 本地桌面端主入口
- Web 仅作为辅助预览或后续扩展入口
- 本地仓库、Git、文件、审查、执行边界由桌面宿主负责

## 二、产品边界

第一版只做本地自用工具，不做：

- 多人在线协作
- 云同步
- 真实模型调用
- 真实 Runner 执行
- 完整权限系统
- 远程部署型后端服务

第一版核心价值是把项目想法、Agent 分工、任务拆解、执行审查、Git checkpoint、知识文档和本地审计收敛到一个桌面应用里。

## 三、技术栈结论

技术栈已定：

```text
桌面宿主 / 本地后端：
Tauri 2 + Rust

前端 UI：
React + TypeScript + Vite + Ant Design

本地数据库：
SQLite

Rust 数据访问：
rusqlite

共享 UI：
React UI 作为唯一界面层，桌面和后续 Web 预览共用同一套 UI

不单独起：
Node.js 原生 HTTP 服务
Fastify
NestJS
FastAPI
```

## 四、技术选择原因

- 这个项目是单人自用，本地文件和 Git 能力更重要，桌面端比浏览器更合适。
- Tauri 2 适合用 Web UI 做界面、用 Rust 做本地能力宿主。
- React + TypeScript + Vite + Ant Design 是成熟、组件化、资料多、AI 生成质量高的一套 UI 组合。
- SQLite 适合单机本地工具，部署简单，数据形态也足够支撑第一版。
- rusqlite 直接、稳定、适合 Rust 本地应用。
- 不单独起后端服务可以减少复杂度，避免为一个本地工具额外维护一层网络服务。
