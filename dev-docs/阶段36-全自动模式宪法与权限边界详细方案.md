# 阶段 36 全自动模式宪法与权限边界详细方案

> 更新提示（2026-06-16）：阶段 36 已完成。本文件中的后续阶段描述以当时路线为准；当前执行顺序以 `dev-docs/当前项目导航.md` 和 `dev-docs/下一步开发路线.md` 为准。当前下一步是阶段 38“项目类型分流与通用想法入口”。

## 1. 背景

当前 `agent-swarm` 已完成阶段 0-35.4：

- 真实模型已通过 Model Gateway、受控模型目录和脱敏审计链路开放。
- 项目计划草案已能进入审批、任务、Runner 请求和执行预览链路。
- Runner 已有 preflight、gate、dry-run、execution lock、minimal run 闸门链。
- 阶段 34 已开放极小范围真实 Runner：仅临时沙箱路径、固定 Git 只读命令、人工二次确认。
- 阶段 35.3/35.4 已把桌面端主控台和项目计划页整理成可操作工作台。

用户后续目标已经从“人工审批驱动”升级为“全自动 Agent 蜂群”：

```text
用户给出想法
  -> 想法引导官扩展想法
  -> 总控智能体选择工作流
  -> 总控智能体给角色智能体授权
  -> 角色智能体调用不同模型和 Skill
  -> 系统自动执行、测试、返修、验收
```

但全自动模式不能等于无边界自由执行。阶段 36 的任务是先把“自动模式下谁能授权、能授权什么、永远不能授权什么、如何停止、如何审计”写清楚并固化到治理文档和设计文档里。

本阶段不急着让 Runner 自动执行。先定宪法、状态机和审计边界。

## 2. 本阶段目标

阶段 36 的目标：

1. 定义 `manual` 与 `full_auto` 两种运行模式。
2. 定义“总控智能体”的最高项目权限边界。
3. 定义自动授权策略：哪些权限可由总控自动授予，哪些必须人工确认，哪些永远禁止。
4. 定义全自动模式的一键停止、暂停、恢复和降级规则。
5. 定义全自动模式必须写入的审计记录。
6. 同步 Agent 宪法、AI 开发细则、维护手册、交接说明和路线图。
7. 为后续阶段 37-43 提供统一边界，避免后面低级智能体各写各的。

## 6. 核心概念定义

### 6.1 运行模式

建议定义两个模式：

```text
manual
full_auto
```

`manual`：

- 默认模式。
- 所有高风险动作仍由用户人工二次确认。
- 阶段 25-35.4 现有行为保持不变。

`full_auto`：

- 用户显式开启。
- 总控智能体可按自动授权策略为角色智能体授予有限权限。
- 所有自动授权、自动调用、自动执行准备和自动执行尝试都必须审计。
- 用户必须可以一键停止。

禁止增加模糊模式名，例如：

```text
smart
fast
free
unlimited
```

### 6.2 总控智能体

总控智能体不是“绕过规则的人”，而是“自动执行规则的人”。

建议定义：

```text
controller_agent
```

职责：

- 读取用户想法和项目目标。
- 选择工作流模板。
- 选择角色智能体。
- 选择模型和 Skill 绑定。
- 根据自动授权策略发放权限。
- 调用 Model Gateway 获取模型输出。
- 创建任务、Runner 请求和执行准备记录。
- 读取测试、构建、执行结果。
- 触发返修或停止。
- 生成交付摘要。

总控智能体禁止：

- 绕过 Model Gateway 发 provider 请求。
- 读取 raw key。
- 返回 raw key、raw prompt、raw response、raw provider error。
- 绕过自动授权策略。
- 绕过审计记录。
- 删除文件。
- 提交或推送 Git。
- 修改受保护路径。
- 发起网络请求。
- 执行自由命令。

### 6.3 角色智能体

角色智能体包括但不限于：

```text
idea_guide
product
architect
frontend
backend
qa
docs
reviewer
security
controller
```

角色智能体只能在总控授予的范围内工作。

禁止：

- 角色智能体自己扩权。
- 角色智能体自己选择未启用模型。
- 角色智能体自己读取 key/base URL。
- 角色智能体绕过审计直接写文件或执行命令。

## 7. 权限分级

建议把权限分成 5 级。

### L0 只读观察

允许：

- 读取项目计划、任务、审批、Runner 请求、审计记录。
- 读取只读预演结果。
- 读取模型目录中允许展示的字段。

禁止：

- 写文件。
- 执行命令。
- 调用真实模型。
- 改 Git。

### L1 模型草案

允许：

- 经 Model Gateway 调用受控模型。
- 写入 `model_calls` 安全审计。
- 生成结构化草案。

要求：

- 必须使用后端模型目录。
- 必须脱敏、截断、落安全审计。
- 不写 raw prompt/response/error。

禁止：

- 自由 provider URL。
- 自由 key。
- 自由 prompt/header/body。

### L2 计划和队列

允许：

- 保存项目计划草案。
- 创建审批记录。
- 创建任务。
- 创建只读 Runner 请求。

禁止：

- 执行 Runner。
- 写项目文件。
- 改 Git。

### L3 执行准备

允许：

- 创建 preflight。
- 创建 gate。
- 创建 dry-run。
- 创建 execution lock。

要求：

- 必须保留完整闸门链。
- 必须校验 scope。
- 必须保持幂等。

禁止：

- 真正执行命令。
- 写用户项目文件。
- 删除文件。
- 改 Git。

### L4 最小真实执行

允许：

- 阶段 34 已有 minimal run 能力。
- 仅允许临时沙箱路径。
- 仅允许固定白名单命令。
- 仅允许人工二次确认后的极小范围执行。

全自动模式第一版不得自动授予 L4。

也就是说，阶段 36 只能定义未来自动授予 L4 的规则，不能现在开放自动 L4。

## 8. 自动授权矩阵

阶段 36 建议写入这个矩阵。

```text
权限级别 | manual 模式 | full_auto 第一版 | 是否可由总控自动授予
L0 只读观察 | 允许 | 允许 | 是
L1 模型草案 | 人工触发 | 允许受控自动 | 是，但必须审计
L2 计划和队列 | 人工审批 | 允许受控自动 | 是，但必须审计
L3 执行准备 | 人工触发 | 允许受控自动 | 是，但必须审计
L4 最小真实执行 | 人工二次确认 | 不自动开放 | 否
Git commit/push | 禁止 | 禁止 | 永不自动
文件删除 | 禁止 | 禁止 | 永不自动
网络请求 | 禁止 | 禁止 | 永不自动
自由命令 | 禁止 | 禁止 | 永不自动
保护路径写入 | 禁止 | 禁止 | 永不自动
```

注意：

阶段 36 只是定规则。后续阶段 42/43 才能讨论是否在非常严格条件下自动进入 L4。

## 9. 一键停止和降级规则

全自动模式必须有一键停止。

建议状态：

```text
enabled
paused
stopping
stopped
failed
```

一键停止触发后：

1. 立即停止创建新的模型调用。
2. 立即停止创建新的 Runner 请求。
3. 立即停止创建新的执行准备记录。
4. 不杀掉已经进入不可中断系统调用的进程，但必须等待其返回并记录结果。
5. 将自动模式状态改为 `stopped` 或 `failed`。
6. 写入停止原因。
7. 前端显示停止状态。

降级规则：

- 任意安全校验失败：`full_auto -> paused`
- 任意审计写入失败：`full_auto -> paused`
- 任意模型输出含敏感值：`full_auto -> paused`
- 任意 scope violation：`full_auto -> stopped`
- 任意保护路径命中：`full_auto -> stopped`
- 任意 Git dirty 状态不符合策略：`full_auto -> paused`

## 10. 审计要求

阶段 36 必须明确：总控智能体拥有最高权限，但不能拥有“无记录权限”。

建议审计对象：

```text
auto_mode_sessions
auto_authorization_records
controller_decisions
agent_delegations
workflow_runs
workflow_step_runs
model_calls
runner_preflight_reviews
runner_execution_gates
runner_dry_runs
runner_execution_locks
runner_minimal_runs
```

第一版可以不全部建表，但文档里必须定义审计原则：

- 谁发起。
- 谁批准。
- 给谁授权。
- 授权级别是什么。
- 授权原因是什么。
- 输入摘要是什么。
- 输出摘要是什么。
- 失败原因是什么。
- 是否触发停止。
- 是否涉及文件。
- 是否涉及命令。
- 是否涉及模型调用。

禁止审计内容：

- raw key。
- raw base URL。
- raw prompt。
- raw provider response。
- raw provider error。
- 用户项目文件完整内容。
- 未脱敏的敏感值。

## 11. 建议数据模型草案

本阶段可以只写入 `docs/data-model-draft.md`，不一定新增 migration。

若后续要落库，建议从这些表开始。

### 11.1 auto_mode_sessions

```sql
CREATE TABLE auto_mode_sessions (
  id TEXT PRIMARY KEY,
  project_id TEXT NOT NULL,
  mode TEXT NOT NULL CHECK (mode IN ('manual', 'full_auto')),
  status TEXT NOT NULL CHECK (status IN ('enabled', 'paused', 'stopping', 'stopped', 'failed')),
  controller_agent_id TEXT NOT NULL,
  started_by TEXT NOT NULL,
  stop_requested_by TEXT,
  stop_reason TEXT,
  policy_version TEXT NOT NULL,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL
);
```

### 11.2 auto_authorization_records

```sql
CREATE TABLE auto_authorization_records (
  id TEXT PRIMARY KEY,
  project_id TEXT NOT NULL,
  auto_mode_session_id TEXT NOT NULL,
  granted_by_agent_id TEXT NOT NULL,
  granted_to_agent_id TEXT NOT NULL,
  permission_level TEXT NOT NULL,
  permission_scope TEXT NOT NULL,
  reason_summary TEXT NOT NULL,
  status TEXT NOT NULL CHECK (status IN ('active', 'revoked', 'expired', 'denied')),
  created_at TEXT NOT NULL,
  revoked_at TEXT
);
```

### 11.3 controller_decisions

```sql
CREATE TABLE controller_decisions (
  id TEXT PRIMARY KEY,
  project_id TEXT NOT NULL,
  auto_mode_session_id TEXT NOT NULL,
  decision_type TEXT NOT NULL,
  input_summary TEXT NOT NULL,
  output_summary TEXT NOT NULL,
  risk_level TEXT NOT NULL,
  status TEXT NOT NULL,
  created_at TEXT NOT NULL
);
```

说明：

- `summary` 字段必须是脱敏摘要。
- 不保存 raw prompt/raw response。
- 真实模型细节仍以 `model_calls` 为准。
