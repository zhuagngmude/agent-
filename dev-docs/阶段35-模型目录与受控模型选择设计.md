# 阶段 35：模型目录与受控模型选择

日期：2026-06-15

## 目标

把当前写死的 gpt-5.4-mini 改成"后端受控模型目录 + 前端下拉选择模型"。用户可以接入中转站里的多个 OpenAI-compatible 文本模型，但不能让前端自由输入 key、base URL、provider、prompt、header、body。

## 关键边界

- 阶段 34 已开放极小范围真实 Runner，但本阶段不改 Runner。
- 不写用户项目文件。不改 Git。不执行命令。不新增自由 provider 配置入口。
- 不把 raw key / raw base URL / raw prompt / raw response / raw provider error 返回前端、日志或 SQLite。
- 不碰保护路径。

## 实现内容

### 1. 后端新增受控模型目录
- Migration `012_add_model_catalog.sql` 创建 `model_catalog` 表。
- 字段：id, project_id, provider, model_id, display_name, purpose, enabled, is_builtin, created_at, updated_at。
- 唯一索引：(project_id, provider, model_id, purpose)。

### 2. 保留安全白名单机制
- 前端只能传 model_catalog 里 enabled=true 的 model_record_id。
- 后端校验 provider=openai_compat, purpose=project_plan_generation, enabled=1, project_id 匹配。

### 3. 修改真实模型调用链
- `ProjectPlanModelDraftInput` 新增可选字段 `model_record_id`。
- `ModelRequest` 新增 `model_id` 字段，从受控目录获取。
- `model_calls.model` 写实际使用的模型名。

### 4. 新增 commands
- `list_project_plan_models` / `update_project_plan_model_enabled`

### 5. 内置模型
- gpt-5.4-mini（默认启用）、gpt-4o-mini、gpt-4.1-mini、gpt-5-mini

### 6. 前端改造
- 模型下拉框，只显示 display_name/model_id/enabled，不展示 key/base URL。
