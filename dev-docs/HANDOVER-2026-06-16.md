## 一、项目概述

Agent Swarm 是一个**桌面端 AI 项目操作系统**。用户输入想法，AI 智能体协作完成项目。

**技术栈：**
- 后端：Tauri 2 + Rust（SQLite 数据库，14 个迁移）
- 前端：React 19 + TypeScript + Vite + Ant Design 6
- AI：DeepSeek API（OpenAI 兼容接口）

**当前阶段：** Stage 38（项目类型分类 + 通用想法入口），但核心执行流程尚未完全打通。

## 四、环境配置

### 必需环境变量

```bash
# DeepSeek API 配置（必须设置，否则 AI 调用会报错）
AGENT_SWARM_OPENAI_COMPAT_API_KEY=<your_api_key_here>
AGENT_SWARM_OPENAI_COMPAT_BASE_URL=https://api.deepseek.com

# 开启真实模型调用（可选，默认 false）
AGENT_SWARM_ENABLE_REAL_MODEL_PROJECT_PLAN=true

# 开启 AI 智能任务生成（可选，默认 false，开启后任务生成会变慢）
AGENT_SWARM_ENABLE_SMART_TASK_GENERATION=true
```

### 启动命令（PowerShell）

```powershell
# 设置环境变量
$env:AGENT_SWARM_OPENAI_COMPAT_API_KEY="<your_api_key_here>"
$env:AGENT_SWARM_OPENAI_COMPAT_BASE_URL="https://api.deepseek.com"
$env:AGENT_SWARM_ENABLE_REAL_MODEL_PROJECT_PLAN="true"

# 启动应用（会编译 Rust + 启动 Vite 前端）
cd F:\Projects\agent-swarm\apps\desktop\src-tauri
cargo tauri dev
```

> ⚠️ 注意：修改 Rust 代码后，`cargo tauri dev` 会自动重新编译，但前端 Vite 需要手动刷新浏览器。

## 七、核心流程说明

用户从**输入想法**到**任务执行**的完整流程：

```
用户输入想法
    ↓
[Stage 38] 想法引导 / 项目类型分类
    ↓
[Stage 39] 项目种子创建
    ↓
[Stage 40] 项目计划生成（AI 或模板）
    ↓
[用户审批] 计划批准 ← 目前需要手动点确认
    ↓
[Stage 41] 任务生成（3-10 个）
    ↓
[自动] 任务分配给 Agents
    ↓
[Stage 42] Runner 执行流程：
    ├── 预检（Preflight Review）
    ├── 放行（Execution Gate）
    ├── 试跑（Dry Run）
    ├── 锁定（Execution Lock）
    └── 真干（Minimal Run）← 这里才真正调 AI 执行任务
    ↓
[Stage 43] 审查交付
```

**当前卡点：** "真干"阶段（`runner_minimal_run.rs`）之前只写记录不调 AI，现已修复但不确定是否完全工作。

## 八、给下一位智能体的建议

### 最高优先级：让应用能完整跑通一个想法

**具体操作步骤：**
1. 关闭正在运行的应用（如果有）
2. 删除数据库：`del F:\Projects\agent-swarm\data\local\agent-swarm.sqlite`
3. 设置环境变量并启动（见第四节）
4. 输入一个简单想法（如"做一个登录页面"）
5. 一路点"确认"，看是否能到达"执行"阶段
6. 如果能执行，检查 DeepSeek 是否真的被调用、生成的内容是否有意义
7. 如果卡住，看终端日志找错误信息

### 已知坑点

1. **`runner_minimal_run.rs` 写文件失败**：本轮已改为创建输出目录；目录创建/文件写入失败会记录为 `io_write_failed`
2. **`auto_swarm.rs` 流程**：预检→放行→试跑→锁定→真干，每一步都可能卡住（需要检查日志）
3. **环境变量未设置时**：应用不会崩溃，但 AI 调用会静默失败，任务内容会是空或错误提示
4. **DeepSeek API 超时**：目前设置 45 秒；复杂任务仍可能不够，后续可再评估是否改为可配置

### 建议的修复顺序

1. ✅ 先让基础流程跑通（删除数据库 → 重启 → 测试）
2. 🔧 修复 `runner_minimal_run.rs` 的错误处理和超时问题
3. ✅ `auto_swarm.rs` 主链路按用户要求转向全自动；后续重点是失败提示、继续做和产物定位
4. 🤖 测试并完善 `generate_smart_tasks_with_ai()`（AI 智能任务生成）
5. 🎨 优化前端 UI（项目计划页、模型配置页）

> 产品目标补充：当前使用者明确希望应用尽量全自动跑通，不希望审批链把核心流程卡成手动操作。后续修改应优先围绕“输入想法 → 自动拆任务 → 自动执行 → 输出结果”的完整体验推进。
