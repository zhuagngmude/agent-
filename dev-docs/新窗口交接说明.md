# 新窗口交接说明

日期：2026-06-19

## 快照

- 项目路径：`F:\Projects\agent-swarm`
- 桌面端主入口：`apps/desktop`
- UI 源码：`packages/ui`
- Runner 输出：`F:\Projects\agent-swarm\workspace\generated`

当前产品方向：个人本地全自动 AI 项目总控台。用户输入目标后，系统自动拆任务、执行 Runner，并把项目产物放到 `workspace/generated`。

P0 AI 员工到 Runner 的核心闭环已经完成：AI 员工配置和项目成员走 Tauri/Rust/SQLite，任务优先绑定 `project_agents.id`，总控分派使用确定性规则，Runner preflight 会调用边界检查，任务页能展示 Agent、执行器、模型和模块范围。

## 当前用户明确要求

- Runner 全面开放，主链路要全自动。
- 不要一堆手动权限确认挡住流程。
- 页面里假东西、没用入口、点不动按钮要清理。
- 任务栏显示任务名字，不显示英文内部状态。
- 做完的项目要能打开文件夹。
- 任务越积越多，所以需要删除。
- 没做完的任务需要继续做。

## 当前主链路

```text
主控台输入目标
-> 总控选择项目 Agent 和专家
-> 任务绑定真实 project_agents.id
-> Runner preflight 做边界检查
-> 任务拆解页查看总任务和 Agent 上下文
-> 继续做 / 打开文件夹 / 删除任务记录
-> workspace/generated 查看项目文件
```

## 当前边界

允许：

- 应用受控服务层内的全自动 Runner 链路。
- 通过 Model Gateway 调用配置好的模型。
- 在 `workspace/generated` 生成项目产物。
- 在系统设置里配置 API Key、Base URL、模型 ID，并写入当前桌面进程环境变量。

禁止：

- 自由 shell。
- Git push/commit。
- 文件删除和保护路径写入，除非用户明确要求且路径已核验。
- 把密钥、Token、raw prompt、raw response、raw provider error 写入文档、日志或数据库。
- 修改 `design/image2/`、`data/local/`、`logs/`、`_internal/`。
