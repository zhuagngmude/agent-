# agent蜂群新窗口交接说明

日期：2026-06-08

## 一句话项目定位

`agent蜂群` 是一个多 AI 智能体协作调度系统，不是普通聊天工具。目标是把一个大项目拆成架构、前端、后端、测试、文档、审查等角色任务，并通过审批、状态管理、知识库和本地 Runner 逐步实现可控的自动协作。

当前阶段先做本地电脑端 Web App，后续再接本地 Runner，最后再考虑封装成桌面应用。

## 如何启动

在项目根目录运行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\start-dev.ps1
```

启动后：

- Mock API: `http://127.0.0.1:8787`
- Web App: `apps\web\index.html`

如果只想启动 API：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\start-mock-api.ps1
```

## 2026-06-10 Agent 权限分层契约接手提示

```text
当前新增 docs/agent-permission-contract.md，用来定义未来“架构 Agent 全权限”或“全部 AI 全权限”的安全含义。

核心结论：
- “全权限”不能是一个 all=true 布尔值。

- architect_admin 可以代表最高规划、编排和申请权限。

不可绕过：
- Approval Service 是高风险授权唯一入口。
- Runner 只能执行已批准 Runner job。
- Agent 不能批准自己的请求。
- 高风险操作仍需人类确认。
- 真实 Runner 写入/Git 操作前仍需 Git checkpoint。
- Model Gateway 真实 provider 请求仍需后续单独 feature flag contract。
- API Key / provider body / prompt / result 仍不得进入前端、SQLite、runtime state 或日志。

本批只是文档契约，不改运行逻辑，不开放真实权限系统，不开放真实 Runner，不开放真实模型调用。
```
