下一阶段目标不是直接把所有功能做完，而是把前端页面变成“可接数据、可接后端、可接 Runner”的工程雏形。

阶段名称：

```text

```

## 应用形态路线

当前看到的是电脑端 Web App，也就是面向电脑大屏使用的网页控制台，不是传统安装版桌面软件。

产品形态分三步：

1. **电脑端 Web App**
   - 当前阶段。
   - 重点是把页面、数据结构、状态机、Runner 审批流程做稳。
   - 直接打开 `frontend/index.html` 或后续通过本地 Web 服务访问。

2. **Web App + 本地 Runner**
   - 推荐的 MVP 正式形态。
   - Web App 负责界面、项目管理、Agent 调度、审批确认、知识库和记录。
   - 本地 Runner 负责读写本地文件、创建 Git checkpoint、生成补丁、跑测试。
   - Runner 不能绕过 Approval Service 自己执行。

3. **桌面应用封装**
   - 后续阶段。
   - 当前 Web App 稳定后，可以用 Tauri 或 Electron 封装为 `agent蜂群.exe`。
   - 桌面应用本质上仍复用 Web App 界面，但启动体验更像普通电脑软件。
   - 不建议现在立刻封装，因为核心流程和数据结构还在稳定。

结论：先做稳电脑端 Web App，再接本地 Runner，最后封装成桌面应用。
